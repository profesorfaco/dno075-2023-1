En Wikipedia se define a la ingeniería inversa o retroingeniería como el proceso llevado para obtener información o un diseño a partir de un producto, con el fin de determinar cuáles son sus componentes y de qué manera interactúan entre sí y cuál fue el proceso de fabricación.

Cuando lo fabricado es código fuente, lo común es usar el término de ingeniería inversa para referir a algo tan "malo" como copiar y pegar, sin siquiera disimular.
